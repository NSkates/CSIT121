Why is it Important?

The first reason why JSON-LD is important is because it gives developers flexibility when making a decision on how they may want their data to be handled. As covered in the About section, XML is better to use if you're planning to design a web application with complex data. If this is not the developer's intention, then it would be redundant to use XML. Possibly making the developer's work more difficult than it needs to be. JSON-LD being the "lighter" option makes developers' jobs easier when handling simple data.

Another reason why it's important is because of its guaranteed compatibility with existing JSON infrastructure. For developers that may want to incorporate and access data from NoSQL document databases, JSON-LD gives them seamless access. Compatability issues can make or break a program. If a program is not compatible with the data it's trying to access, then the program will not be able to access the data at all. This is why JSON-LD's compatibility with JSON is so important.

Finally, JSON-LD " is ideal for transferring data between different systems and programming languages" (Oracle, n.d.). Especially nowadays, data isn't always stored in one place. Sometimes, it may need to be transffered to another system or be copied and stored in multiple places at once. Since it's not limited to one language, a developer won't need to go through the trouble on "translating" the data to be compatible with a different syntax.
